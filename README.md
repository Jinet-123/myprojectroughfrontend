# Propx – Full Stack Real Estate Marketplace (MERN)

Propx is a full-stack MERN real estate marketplace featuring multi-role authentication (User, Seller, Admin), seller verification workflow, and analytics dashboard. Users can browse and enquire about properties, sellers manage property listings and buyer enquiries, and admins control platform access and monitor activity.

---

## 🚀 Live Application
Frontend: https://myprojectroughfrontend.vercel.app/  
Backend API: https://myprojectrough-1.onrender.com  

---

## ✨ User Features
- Browse and search listed properties
- View detailed property information
- Add properties to wishlist
- Send enquiries to property sellers
- Edit and manage user profile
- Request to become a seller (subject to admin approval)

---

## 🏷 Seller Features
- Dedicated Seller Dashboard
- Add new property listings
- Edit property details
- Mark properties as sold
- Delete own property listings
- Manage and respond to buyer enquiries
- Track personal property listings

---

## 👑 Admin Features
- Approve or reject seller requests
- View and manage platform users
- Monitor all listed properties
- Dashboard analytics:
  - Total properties
  - Property category distribution
  - Total users and sellers
  - Platform activity overview

---

## 🛠 Tech Stack

Frontend:
- React
- Vite
- Tailwind CSS
- Axios

Backend:
- Node.js
- Express.js
- MongoDB
- JWT Authentication
- Render(Deployment)

## 🔐 Core Functionalities
- Role-based authentication (User / Seller / Admin)
- Protected routes for each role
- Seller approval workflow
- Property management system (CRUD operations)
- Enquiry management system
- RESTful API architecture
- Secure JWT-based authorization

## ⚙️ Installation (Local Setup)

Clone repository:

# Clone Frontend 
git clone https://github.com/Jinet-123/myprojectroughfrontend  

# Clone Backend
git clone https://github.com/Jinet-123/myprojectrough 

### 🔹 Backend Setup

cd myprojectrough  
npm install  
npm start  

### 🔹 Frontend Setup

cd myprojectroughfrontend  
npm install  
npm run dev  

---

Create `.env` file inside frontend project:
